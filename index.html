Custom Favicon Implementation

Learn how to add a custom favicon to replace the Firebase default

What is a Favicon?

A favicon (favorite icon) is a small image displayed next to the page title in the browser tab. It helps users identify your website easily when they have multiple tabs open.

How to Change Your Favicon

1

Create your favicon - Use a 64x64 or 32x32 pixel image. PNG format is recommended for quality.

2

Add the favicon to your project - Place the image file in your project's public folder or root directory.

3

Add the HTML code - Include the following code in the <head> section of your HTML file:

<!-- Favicon --> <link rel="icon" type="image/x-icon" href="/favicon.ico"> <link rel="icon" type="image/png" href="/favicon.png"> <!-- For Apple devices --> <link rel="apple-touch-icon" href="/apple-touch-icon.png"> <!-- For Android devices --> <link rel="icon" type="image/png" sizes="192x192" href="/android-chrome-192x192.png"> <link rel="icon" type="image/png" sizes="512x512" href="/android-chrome-512x512.png"> <!-- Web App Manifest --> <link rel="manifest" href="/site.webmanifest">
4

Clear your browser cache - Old favicons are often cached by browsers. Clear your cache or do a hard refresh (Ctrl+F5) to see the changes.

5

Redeploy your site - If you're using Vercel, push your changes to GitHub to trigger a new deployment.

Best Practices

• Use a simple design that works well at small sizes

• Save your favicon in multiple formats and sizes for different devices

• Use a square aspect ratio for your favicon

• Test your favicon on different browsers and devices
